# ContextSystem — AI エージェント向け利用仕様（正）

クローン直後に**このファイルを最優先で読む**こと。  
Cursor・OpenAI Codex・Claude Code・GitHub Copilot など、ツールの違いに依存せず、ここを単一の仕様として扱う。

## このリポジトリの役割

コンテキスト（文脈・エージェントに渡す情報の設計など）に関する知見と実験を置く。  
ルート直下の**各ディレクトリは情報スコープが閉じた単位**であり、原則として**そのディレクトリ単体で利用可能**である。

## 現状：何が使えるか / 使えないか

### 使えること

| 対象 | できること | やり方 | 結果 |
|------|------------|--------|------|
| リポジトリ全体 | 設計原則・エージェント向け仕様の確認 | **ルート `AGENTS.md` を必ず読む**（必要なら `README.md` も）。ツール固有設定は使わない | スコープ境界・可否・手順・更新ルールが分かる |
| ルート直下の各ディレクトリ（追加後） | そのスコープ内の手順・実験の実行 | 当該ディレクトリの `AGENTS.md`（または `README.md`）を読み、記載の手順に従う | 各スコープが約束する成果物・ログ・検証結果が得られる |

### 使えないこと（現状）

| 対象 | 理由 |
|------|------|
| ルート直下以外からの横断自動化 | 設計上、ルートの各ディレクトリは相互に前提を持たない |
| 未記載の「暗黙の手順」 | 手順は必ず文書化する。文書にないことはサポート対象外として扱う |

※ ルート直下に実験用ディレクトリが増えたら、**下の「ルート直下スコープ一覧」表を必ず更新する**。

## ルート直下スコープ一覧（エージェント用インデックス）

| ディレクトリ | 単体利用の入口 | いまできること（要約） |
|--------------|----------------|-------------------------|
| （未追加） | — | 実験ディレクトリを追加したら本表と当該 `AGENTS.md` を更新すること |

新規ディレクトリ追加時は、最低限次を満たすこと。

1. 上表に1行追加する。
2. そのディレクトリ内に **`AGENTS.md`** を置き、次を同じ表形式で書く：  
   **使える / 使えない / できること / どうやるか / 何が結果か**  
   （そのスコープ内だけで完結する内容に限る。）

## 全エージェントが同じ情報を見る方法（ツール設定に依存しない）

このリポジトリには **Cursor / Codex / Claude Code / GitHub Copilot など、特定プロダクト専用の設定ファイルを置かない**（パス・ファイル名が各社仕様変更で無効化されるのを避けるため）。

- **仕様の正本は常に本ファイル（ルート `AGENTS.md`）だけ**とする。別ツール用の入口ファイルを増やして二重記述しない。
- 利用者・オーケストレーターは、**どのエージェントでも同じ操作**でよいようにする：  
  例）リポジトリルートの `AGENTS.md` を開いて読ませる／チャットに添付する／「まず `AGENTS.md` を読んでから作業」と指示する／`@AGENTS.md` のように参照させる（各ツールの添付・@ の書き方はツール側に従うが、**指す先のパスは常にこの1ファイル**）。
- サブスコープの詳細は、当該ディレクトリの `AGENTS.md` に書き、ルートの「ルート直下スコープ一覧」から辿る。

これにより、特定エージェントの「リポジトリ内のお約束ファイル名」が変わっても、**参照先のパスを `AGENTS.md` に合わせるだけ**で切り替えられる。

## メンテナンス契約（システムとしてのルール）

- **仕様の正本**はルートの `AGENTS.md`（および各スコープ内 `AGENTS.md`）のみ。ツール専用の重複ファイルは追加しない。
- 利用可否・手順・期待結果が変わったら、**同じコミットで**本ファイル（と該当スコープの `AGENTS.md`）を更新する。
- エージェントは「文書に書かれたことのみ」を実行可能な能力として扱い、推測で横断結合しない。
