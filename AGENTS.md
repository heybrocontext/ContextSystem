# ContextSystem — AI エージェント向け利用仕様（正）

**正本の実体パス（いま）**: リポジトリルートのこのファイル `AGENTS.md`。  
**読み方**: 各ツールがこのファイルを**自動で読む保証はない**（Cursor / Codex / Claude Code / Copilot で挙動は違う）。だからこそ、毎回「このパスをコンテキストに**参照させる**」（開く・添付・プロンプトでパス指定など）ことが前提になる。

**ファイル名について**: `AGENTS.md` は「どこかのプロダクトの標準だから正しい」というより、**このリポジトリが現時点で正本を置いているパス**である。業界で別名に移る・特定ツールが別ファイルを推すようになっても、**正本の中身はこの1か所に集約する方針は変えず**、必要ならファイル名だけ変更して `README.md` と本書の自己参照・ルート直下表を更新すればよい（ツールの暗黙のお約束には依存しない）。

## このリポジトリの役割

コンテキスト（文脈・エージェントに渡す情報の設計など）に関する知見と実験を置く。  
ルート直下の**各ディレクトリは情報スコープが閉じた単位**であり、原則として**そのディレクトリ単体で利用可能**である。

## 現状：何が使えるか / 使えないか

### 使えること

| 対象 | できること | やり方 | 結果 |
|------|------------|--------|------|
| リポジトリ全体 | 設計原則・エージェント向け仕様の確認 | **ルート正本をコンテキストに参照させてから読む**（現状のパスは `AGENTS.md`。必要なら `README.md` も）。自動読込に頼らない。ツール固有設定は使わない | スコープ境界・可否・手順・更新ルールが分かる |
| ルート直下の各ディレクトリ（追加後） | そのスコープ内の手順・実験の実行 | 当該ディレクトリの `AGENTS.md`（または `README.md`）を読み、記載の手順に従う | 各スコープが約束する成果物・ログ・検証結果が得られる |

### 使えないこと（現状）

| 対象 | 理由 |
|------|------|
| ルート直下以外からの横断自動化 | 設計上、ルートの各ディレクトリは相互に前提を持たない |
| 未記載の「暗黙の手順」 | 手順は必ず文書化する。文書にないことはサポート対象外として扱う |

## 全エージェントが同じ情報を見る方法（ツール設定に依存しない）

このリポジトリには **Cursor / Codex / Claude Code / GitHub Copilot など、特定プロダクト専用の設定ファイルを置かない**（パス・ファイル名が各社仕様変更で無効化されるのを避けるため）。

- **仕様の正本は常にルートのこの1ファイルだけ**（現状のファイル名は `AGENTS.md`）。別ツール用の入口ファイルを増やして二重記述しない。
- 利用者・オーケストレーターは、**どのエージェントでも「正本パスを明示的に渡す」**のが共通手順になる：  
  例）ルートの正本を開いて読ませる／添付する／「まずリポジトリルートの `AGENTS.md` を読んでから」とプロンプトに書く／`@AGENTS.md` 等で参照（**@ の有無・自動読込の有無はツール依存**。ここでは**人が渡すパスを揃える**ことで全員同じ情報を見せる）。
- サブスコープの詳細は、当該ディレクトリの `AGENTS.md` に書き、ルートの「ルート直下スコープ一覧」から辿る。サブ側のファイル名も慣習であり、スコープ内で完結するならリネーム可（その場合は一覧と当該文書の相互参照を更新）。

**腐らせないために**: 「Codex が `AGENTS.md` を標準として拾うから全員読める」という前提は置かない。**参照で読ませる**のが本体。ファイル名の慣習が変わったら、**渡すパスと README・本書の先頭の「正本の実体パス」表記を更新する**だけで追随できる。

## メンテナンス契約（システムとしてのルール）

- **仕様の正本**はルートの1ファイル（現状 `AGENTS.md`）および各スコープ内の正本（現状各 `AGENTS.md`）のみ。ツール専用の重複ファイルは追加しない。正本をリネームしたら **README・本ファイル先頭・ルート直下表・該当スコープ** のパス表記を同じコミットで揃える。
- 利用可否・手順・期待結果が変わったら、**同じコミットで**本ファイル（と該当スコープの正本）を更新する。
- エージェントは「文書に書かれたことのみ」を実行可能な能力として扱い、推測で横断結合しない。
