# ContextSystem — AI エージェント向け利用仕様（正）

**仕様本文の正本**: このファイル（リポジトリルートの `CONTEXTSYSTEM_SPEC.md`）。  
ルートの **`AGENTS.md` は本文を持たない入口**であり、ツールが慣習的にそちらだけを開く場合でも、**必ず本ファイルまで読みに来る**こと。

**読み方**: 各ツールが `AGENTS.md` や本ファイルを**自動で読む保証はない**（Cursor / Codex / Claude Code / Copilot で挙動は違う）。だからこそ、毎回「**本ファイル**（または入口経由で本ファイル）をコンテキストに**参照させる**」（開く・添付・プロンプトでパス指定など）ことが前提になる。

**ファイル名について**: `CONTEXTSYSTEM_SPEC.md` は**このリポジトリが仕様本文を置くパス**である。`AGENTS.md` は業界やプロダクトの慣習に合わせた**薄い入口**に過ぎない。いずれの慣習が変わっても、**本文は `CONTEXTSYSTEM_SPEC.md` に集約する方針は変えず**、必要なら「入口ファイル名・本ファイル名」を変えて **`README.md`・入口・本ファイル先頭** のパス表記を同じコミットで揃える（ツールの暗黙のお約束には依存しない）。

## このリポジトリの役割

コンテキスト（文脈・エージェントに渡す情報の設計など）に関する知見と実験を置く。  
ルート直下の**各ディレクトリは情報スコープが閉じた単位**であり、原則として**そのディレクトリ単体で利用可能**である。

## 現状：何が使えるか / 使えないか

### 使えること

| 対象 | できること | やり方 | 結果 |
|------|------------|--------|------|
| リポジトリ全体 | 設計原則・エージェント向け仕様の確認 | **`CONTEXTSYSTEM_SPEC.md` をコンテキストに参照させてから読む**（必要なら `README.md` も）。入口の `AGENTS.md` しか開いていない場合は、そこから本ファイルへ進む。自動読込に頼らない。ツール固有設定は使わない | スコープ境界・可否・手順・更新ルールが分かる |
| ルート直下の各ディレクトリ（追加後） | そのスコープ内の手順・実験の実行 | 当該ディレクトリの `AGENTS.md`（または `README.md`）を読み、記載の手順に従う | 各スコープが約束する成果物・ログ・検証結果が得られる |

### 使えないこと（現状）

| 対象 | 理由 |

| 未記載の「暗黙の手順」 | 手順は必ず文書化する。文書にないことはサポート対象外として扱う |

## 全エージェントが同じ情報を見る方法（ツール設定に依存しない）

このリポジトリには **Cursor / Codex / Claude Code / GitHub Copilot など、特定プロダクト専用の設定ファイルを置かない**（パス・ファイル名が各社仕様変更で無効化されるのを避けるため）。

- **仕様の本文は `CONTEXTSYSTEM_SPEC.md` のみ**に書く。ルートの `AGENTS.md` は本ファイルへの参照だけにし、本文の二重記述をしない。
- 利用者・オーケストレーターは、**どのエージェントでも「本ファイルのパスを明示的に渡す」**のが共通手順になる：  
  例）`CONTEXTSYSTEM_SPEC.md` を開いて読ませる／添付する／「まずリポジトリルートの `CONTEXTSYSTEM_SPEC.md` を読んでから」とプロンプトに書く／`@CONTEXTSYSTEM_SPEC.md` 等で参照（**@ の有無・自動読込の有無はツール依存**。ここでは**人が渡すパスを揃える**ことで全員同じ情報を見せる）。
- サブスコープの詳細は、当該ディレクトリの `AGENTS.md` に書き、ルートの「ルート直下スコープ一覧」から辿る。サブ側のファイル名も慣習であり、スコープ内で完結するならリネーム可（その場合は一覧と当該文書の相互参照を更新）。

**腐らせないために**: 「`AGENTS.md` をツールが拾うから全員読める」という前提は置かない。**`CONTEXTSYSTEM_SPEC.md` を参照で読ませる**のが本体。入口やファイル名の慣習が変わったら、**渡すパスと README・本ファイル先頭・入口 `AGENTS.md` のリンク**を更新するだけで追随できる。

## メンテナンス契約（システムとしてのルール）

- **仕様本文の正本**はルートの `CONTEXTSYSTEM_SPEC.md`、および各スコープ内の正本（現状各 `AGENTS.md`）のみ。ツール専用の重複ファイルは追加しない。`CONTEXTSYSTEM_SPEC.md` または入口 `AGENTS.md` をリネームしたら **README・本ファイル先頭・入口・ルート直下表・該当スコープ** のパス表記を同じコミットで揃える。
- 利用可否・手順・期待結果が変わったら、**同じコミットで**本ファイル（と該当スコープの正本）を更新する。
- エージェントは「文書に書かれたことのみ」を実行可能な能力として扱い、推測で横断結合しない。
