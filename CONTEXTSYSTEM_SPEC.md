# ContextSystem — AI エージェント向け利用仕様（正）

## 正本・入口・リネーム

- **仕様本文の正本**はこのファイル（`CONTEXTSYSTEM_SPEC.md`）のみ。ルートの **`AGENTS.md` は本文を持たない入口**（`AGENTS.md` しか開いていない場合は、必ず本ファイルまで読む）。
- 各ツールが `AGENTS.md` や本ファイルを**自動で読む保証はない**。毎セッション、**本ファイルをコンテキストに参照させる**（開く・添付・プロンプトでパス指定・`@CONTEXTSYSTEM_SPEC.md` 等。ツールの書き方に従う）。
- このリポジトリには **Cursor / Codex / Claude Code / GitHub Copilot など、プロダクト専用の設定ファイルを置かない**。
- ファイル名を変えるときは **`README.md`・`AGENTS.md` のリンク先・本節・ルート直下表・該当スコープ** のパス表記を、同じコミットで揃える。

## このリポジトリの役割

コンテキスト（文脈・エージェントに渡す情報の設計など）に関する知見と実験を置く。  
ルート直下の**各ディレクトリは情報スコープが閉じた単位**であり、原則として**そのディレクトリ単体で利用可能**である。

クローン後にエージェントが把握すべき解像度は、**いま何が使えるか／使えないか／使える範囲で何ができるか／どうやるか／何が結果か**である（下表および各スコープの `AGENTS.md` で同じ軸を埋める）。

## 現状：何が使えるか / 使えないか

### 使えること

| 対象 | できること | やり方 | 結果 |
|------|------------|--------|------|
| リポジトリ全体 | 設計原則・エージェント向け仕様の確認 | **本ファイルをコンテキストに参照させてから読む**（必要なら `README.md` も）。入口の `AGENTS.md` しか開いていない場合は、そこから本ファイルへ進む。自動読込に頼らない | スコープ境界・可否・手順・更新ルールが分かる |
| ルート直下の各ディレクトリ（追加後） | そのスコープ内の手順・実験の実行 | 当該ディレクトリの `AGENTS.md`（または `README.md`）を読み、記載の手順に従う | 各スコープが約束する成果物・ログ・検証結果が得られる |

### 使えないこと（現状）

| 対象 | 理由 |
|------|------|
| ルート直下以外からの横断自動化 | 設計上、ルートの各ディレクトリは相互に前提を持たない |
| 未記載の「暗黙の手順」 | 手順は必ず文書化する。文書にないことはサポート対象外として扱う |

※ ルート直下に実験用ディレクトリが増えたら、**下の「ルート直下スコープ一覧」表を必ず更新する**。

## メンテナンス契約（システムとしてのルール）

- **仕様本文**はルートの `CONTEXTSYSTEM_SPEC.md`、および各スコープ内の正本（現状各 `AGENTS.md`）のみ。ツール専用の重複ファイルは追加しない。ルートの入口 `AGENTS.md` には本文を足さない。
- 利用可否・手順・期待結果が変わったら、**同じコミットで**本ファイル（と該当スコープの正本）を更新する。
- エージェントは「文書に書かれたことのみ」を実行可能な能力として扱い、推測で横断結合しない。
