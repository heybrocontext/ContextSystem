# ContextSystem

コンテキスト（文脈・エージェントやツールに渡す情報の設計など）に関する**知見**と**実験**を置くリポジトリです。

## アーキテクチャの最重要事項（エージェント共通）

Cursor・OpenAI Codex・Claude Code・GitHub Copilot など、**どのエージェントがこのリポジトリをクローンして解析しても**、次を同じ解像度で認識できるように設計する（リポジトリを**ひとつのシステム**として扱う）。

1. **現状、何が使えるか / 使えないか**
2. **使える範囲では、何ができるか**
3. **どうやるか**（手順は文書化されたものに限定）
4. **何が結果か**（期待する出力・成果物）

そのための**仕様本文の正本**はルートの **`CONTEXTSYSTEM_SPEC.md`** に1か所だけ集約する。ツールごとの設定ファイルは置かない。  
ルートの **`AGENTS.md`** は慣習的に拾われることがある**入口**にすぎず、本文は持たず **`CONTEXTSYSTEM_SPEC.md` を指すだけ**とする。

人間向けの概要は本 README、可否・手順・結果・更新ルールはすべて **`CONTEXTSYSTEM_SPEC.md`** に書く。

**全ツールが自動で同じファイルを読むわけではない**。**毎セッション、`CONTEXTSYSTEM_SPEC.md` をコンテキストに参照させる**（開く・添付・プロンプトでパス指定など）ことを前提にする。ファイル名は慣習に合わせて変えてもよいが、その場合は **README・`AGENTS.md`・`CONTEXTSYSTEM_SPEC.md` 先頭** のパス表記を同じコミットで揃える。

## 設計原則

- **ルート直下の各ディレクトリ**がひとつの単位です。
- 各ディレクトリは**情報スコープがそこで閉じる**ようにします（他ディレクトリの前提を読まなくてよい）。
- 各ディレクトリは**単体で利用できる**ようにドキュメント・コード・設定を揃えます。

トップレベルに新しい試みやノートを追加するときは、この原則に沿って切り出し、**`CONTEXTSYSTEM_SPEC.md` の「ルート直下スコープ一覧」と各スコープ内 `AGENTS.md`** を更新してください。
